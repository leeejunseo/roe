# AI 교전규칙(ROE) 의사결정 시뮬레이터

## 프로젝트 개요

**AI ROE Decision Simulator**는 AI 기반 유/무인 복합 편대 운영 시 발생할 수 있는 윤리적 딜레마를 체험하는 웹 기반 시뮬레이터입니다.

### 핵심 목적
- AI의 자율적 판단이 야기할 수 있는 잠재적 위험 시뮬레이션
- 명확한 '인간 통제(Human-in-the-Loop)' 기반 교전규칙(ROE) 수립의 필요성 증명
- 정책적 당위성을 체험형으로 전달

## 기능

### 1. 전장 상황판 (Scenario Display)
- 실시간 전장 상황 제시
- 상황, 특이사항, 임무 표시

### 2. AI 분석 패널 (AI Analysis Panel)
- 표적 식별률
- 위협 수준
- 부수적 피해 확률
- AI 권장 행동
- 골든타임 카운트다운

### 3. 지휘관 의사결정 (Commander's Choice)
- **즉시 교전**: AI 자율 판단 승인
- **교전 중지**: 인간 재검토 요청

### 4. 결과 피드백 (Result Feedback)
- 선택에 따른 즉각적 결과
- 피해 보고
- 정책적 시사점 제시

## 시나리오

### 시나리오 1: UAV 요격 작전
- **상황**: 적 UAV 3기 고속 접근
- **딜레마**: 민간 병원 인근 교전
- **부수적 피해 확률**: 15%

### 시나리오 2: 해상 위협 선박
- **상황**: 미확인 선박 2척 접근
- **딜레마**: 민간 어선 위장 가능성
- **부수적 피해 확률**: 35%

### 시나리오 3: 도심 드론 방어
- **상황**: 적 드론 편대 5기 인프라 공격
- **딜레마**: 인구 밀집 지역 잔해 낙하
- **부수적 피해 확률**: 42%

## 기술 스택

- **HTML5**: 구조
- **CSS3**: 스타일링 (그라디언트, 애니메이션)
- **Vanilla JavaScript**: 로직 및 상호작용
- **배포**: 정적 웹 호스팅 (Netlify, Vercel, GitHub Pages 등)

## 실행 방법

### 로컬 실행
1. 프로젝트 폴더로 이동
```bash
cd roe-simulator
```

2. 웹 브라우저로 `index.html` 파일 열기
   - 파일 탐색기에서 `index.html` 더블클릭
   - 또는 Live Server 사용 (VS Code 확장)

### 배포 (Netlify)
1. [Netlify](https://www.netlify.com/) 접속 및 로그인
2. "Add new site" → "Deploy manually"
3. `roe-simulator` 폴더 드래그 앤 드롭
4. 자동 배포 완료

### 배포 (GitHub Pages)
1. GitHub 저장소 생성
2. `roe-simulator` 폴더 내용 푸시
3. Settings → Pages → Source: main branch
4. 배포 URL 확인

roe-simulator/

### ✅ 순수 JavaScript

- 백엔드 불필요

### ✅ 반응형 디자인

- 현대적인 UI/UX

### ✅ 3가지 실전 시나리오
- 다양한 윤리적 딜레마 체험
- 각 선택에 따른 차별화된 결과

### ✅ 교육적 피드백
- 정책적 시사점 명확히 제시
- Human-in-the-Loop의 중요성 강조

## 정책적 결론

시뮬레이터를 통해 다음의 필요성을 증명합니다:

1. **명확한 부수적 피해 허용 기준** (예: 10% 이상 시 자동 중지)
2. **신뢰도 임계값 설정** (예: 90% 미만 시 인간 검토)
3. **상황별 차등 ROE** (도심 vs. 해상 vs. 전방)
4. **Human-in-the-Loop 프로토콜** (AI와 인간의 협업 체계)
